■概要
  Access ALAYAはAccessをマルチユーザで安全に安定して使うためのフレームワークです。

  Access は Excel のような手軽さでデータベースを扱えるため、DBサーバを立てるまでもない
  小規模のデータベースを使いたいときに威力を発揮します。ところがAccessはスタンドアロン
  もしくは多くても２～３人のごく少人数のユーザしか想定していないため、デフォルト設定の
  まま多人数で操作すると簡単に壊れてしまう脆弱性も抱えています。

  Access ALAYAはこの脆弱性を回避して、数十人規模の同時使用でも安全にAccessを運用する
  ことができるようになります。
  
  部内や課内などのグループで共有台帳を運用するには十分なユーザ数でしょう。作者の
  システムでは登録ユーザ100名、同時ログイン30名程度の環境で実用的に動作しており、
  ノーメンテで1年以上にわたって安定して稼働しています。
  
  Access ALAYAは以下の機能を実現します。

【マルチユーザ対応】
  最初からマルチユーザ運用を想定してます。ALAYAをプラットフォームとしたシステムでは
  後になってマルチユーザ向けのチューニングを行うような必要はありません。

【ローコード開発】
  共通関数をライブラリ化してできるだけコードを書かずに済むようにしています。ユーザ
  ロジックに集中できるのでコードの見通しが良くなり、メンテナンス性も高まります。

【バージョンアップと配布管理】
  エンドユーザ向け機能にバージョンアップは欠かせません。ALAYAは頻繁なバージョンアップと
  ユーザ配布が行われることを前提としており、モジュール更新とバージョン管理、エンド
  ユーザへの配布の仕組みを用意しています。

■インストール
  本ソフトはファイルサーバに配置して動作します。
  上記のALAYAインストールフォルダにある「ALAYA Installer**.accdb」を起動してサーバに
  インストールして下さい。
  バージョン番号の一番大きなものが最新のインストーラです。
  
■アンインストール
  レジストリ登録などは行ってないので、ファイルサーバから展開したファイルを削除する
  だけでアンインストールされます。
  サーバにコピーされた「プロジェクト名」配下を削除してください。

■使用方法
  ALAYA起動はスクリプト経由で行います。
  メニューモジュールフォルダに起動用のVBSスクリプトが生成されるので、これを
  ダブルクリックして起動してください。
  クライアントへはGUIモジュール(.accdb)ではなく、サーバに配置したスクリプトの
  ショートカットを配布してください。
